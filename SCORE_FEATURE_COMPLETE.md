## 📊 数据库结构

### students表（成绩相关字段）
| 字段名 | 类型 | 说明 |
|--------|------|------|
| score | INTEGER | 高考总分（0-750） |
| rank | INTEGER | 省排名 |
| subject_scores | JSONB | 各科成绩JSON |

### subject_scores JSON结构
```json
{
  "chinese": 120,    // 语文（0-150）
  "math": 135,       // 数学（0-150）
  "english": 130,    // 英语（0-150）
  "subject1": 85,    // 选科1（0-100）
  "subject2": 90,    // 选科2（0-100）
  "subject3": 88     // 选科3（0-100）
}
```

## 💡 设计优势

### 1. 使用JSONB的好处
- ✅ 灵活：可以适应不同省份的选科组合
- ✅ 可扩展：未来可以添加更多科目
- ✅ 可查询：PostgreSQL支持JSONB查询
- ✅ 节省空间：不需要为每个科目创建单独字段

### 2. 数据完整性
- ✅ 前端必填验证
- ✅ 分数范围验证
- ✅ 总分与各科成绩可以交叉验证

### 3. 用户体验
- ✅ 默认展开，减少操作步骤
- ✅ 清晰的必填标识
- ✅ 实时验证反馈
- ✅ 友好的错误提示
