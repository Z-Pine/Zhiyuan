## 架构设计

### 后端架构
```
┌─────────────────────────────────────────┐
│              API Gateway                │
│         (Express + Middleware)          │
├─────────────────────────────────────────┤
│  Auth  │  Student │ Score │ Recommendation │
│ Module │  Module  │Module │    Module      │
├─────────────────────────────────────────┤
│         Service Layer                   │
│  (Business Logic + Algorithm)           │
├─────────────────────────────────────────┤
│         Data Access Layer               │
│       (PostgreSQL + pg driver)          │
└─────────────────────────────────────────┘
```

### 前端架构
```
┌─────────────────────────────────────────┐
│           Presentation Layer            │
│         (Pages + Widgets)               │
├─────────────────────────────────────────┤
│           State Management              │
│           (Provider)                    │
├─────────────────────────────────────────┤
│           Service Layer                 │
│     (API Service + Storage)             │
├─────────────────────────────────────────┤
│           Data Layer                    │
│  (HTTP Client + Local Storage)          │
└─────────────────────────────────────────┘
```

## 技术选型理由

### 为什么选择Node.js + Express？
1. **开发效率高**: JavaScript全栈开发
2. **生态丰富**: npm包管理器有大量可用库
3. **性能优秀**: 异步非阻塞I/O适合高并发
4. **易于部署**: 云原生支持好

### 为什么选择PostgreSQL？
1. **功能强大**: 支持复杂查询、JSON、全文搜索
2. **稳定性高**: 生产环境验证的数据库
3. **免费开源**: 无商业授权费用
4. **云支持好**: Neon等平台提供免费托管

### 为什么选择Flutter？
1. **跨平台**: 一套代码支持Web、iOS、Android
2. **性能优秀**: 接近原生性能
3. **UI一致**: 统一的设计语言
4. **热重载**: 开发效率高

### 为什么选择Provider？
1. **轻量级**: 不需要复杂的学习曲线
2. **官方推荐**: Flutter团队推荐的状态管理方案
3. **易于测试**: 状态与UI分离
4. **性能优秀**: 精准的状态更新
