## 7. 测试管理

### 7.1 测试流程

```
┌─────────────────────────────────────────────────────────────────┐
│                       测试流程                                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  需求评审 ── 用例编写 ── 用例评审 ── 执行测试 ── 缺陷管理 ── 上线 │
│    │           │          │          │          │        │      │
│    ▼           ▼          ▼          ▼          ▼        ▼      │
│  需求冻结   测试用例   用例覆盖率   执行率100%   Bug关闭  通过率   │
│             ≥90%                   + 修复率≥95%           ≥95%   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 7.2 缺陷等级

| 等级 | 定义 | 处理时限 | 示例 |
|------|------|---------|------|
| P0-Bug | 功能不可用/数据错误 | 24h内修复 | 登录崩溃、推荐结果错误 |
| P1-Bug | 功能异常/影响使用 | 3天内修复 | 页面加载慢、交互异常 |
| P2-Bug | 界面问题/体验不佳 | 5天内修复 | 样式错位、文字溢出 |
| P3-Bug | 建议性问题 | 迭代处理 | 加载动画优化 |

### 7.3 测试工具

| 用途 | 工具 | 说明 |
|------|------|------|
| 接口测试 | Postman / Apifox | API自动化测试 |
| 性能测试 | JMeter / k6 | 压力测试 |
| 自动化测试 | Jest / Flutter Test | 单元测试 |
| 抓包工具 | Charles / Fiddler | 网络调试 |
| 设备云测 | WeTest / Testin | 兼容测试 |
